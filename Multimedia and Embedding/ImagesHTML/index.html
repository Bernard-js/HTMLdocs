the img tag also has an alternative text attribute, alt, which is pretty usefull for cases where the image could not be rendered on the page.

There is many reasons why you'd like to have an alt attribute to your image:

  • Screen reader users can listen to the text and have a brief description of the image.

  • as was shown in the example above, the path name can be misspelled.

  • The image type isn't supported by the browser.

  • Search engines can use the text in search queries.

  • Users have turned off images to reduce data transfer volume.

Width and Height

it's also possible to specify the width and height of the image. this helps, for example, when the page hasn't yet been loaded, the browser sets a space to fill the image whem it's fully loaded. This makes the loading process smoother, improving the user experience.

However, you shouldn't change the image's width and height at the attribute. It distorts the image natural form and doesn't produce a good user experience. you should use a proper image editor, like adobe photoshop, or CSS instead.

Image Titles

you can add the title attribute,that gives a brief image description when the mouse is hovered over the image.

However, it's not recommended to do it, mainly because of many accessibility problems that have occured with screen readers. you should put that informatio in the article's text instead.

Annotating images with figures and captions

you can put notes and caption to an image by creating a div containing the image and the caption, and then style with CSS and do whatever you want.

That's fine, but it isn't recommended, as it has no semantic value. Instead the HTML5 tags <figure> and <figcaption> fits perfectly at this context.

CSS background images

You can also use CSS to embed images into webpages by using the CSS background-image property For example, to place a background image on every paragraph on a page, you could do this: 

                    p {
                        background-image: url("images/dinosaur.jpg");
                      }

dealing with embedded CSS images is a lot easier to position and control then with HTML ones. So why use HTML instead? CSS images doesn't have any semantic meaning at all, it has no text equivalent, and cannot be read by screen readers, so it's recommended for decoration purposes only.

Summing up: if an image has meaning, in terms of your content, you should use an HTML image. If an image is purely decoration, you should use CSS background images.
