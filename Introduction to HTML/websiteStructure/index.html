Document and Website Structure

Basic Sections of a Document

  • Header: <header>

  • Navigation Bar: <nav>

  • Main Content: <main>, with various content subsctions represented by <article>, <section> and <div> elements.

  • Sidebar: <aside>; usually placed inside <main>.

  • Footer: <footer>

Section Elements definition and good practices

semantic sectioning elements

<main> is for content just for this page. Main element must be a direct child of body, nesting it with other elements is a bad practice.

<article> it encloses a piece of content that has a meaning on his own, independing of other contents of the same page.

<section> it groups one piece of page that execute a single task. is a good practice to start each section with a heading. It is possible to nest articles in a section and sections in an article, it depends on each case.

<aside> contains contents that are not directly related to the main content, but it can have additional information indirectly related with the main content(like related links, author bio, etc...).

<header> represents a group of introductory content. If it is a child of <body> it defines the global header of a webpage, but if it's a child of an <article> or <section>it defines a specific header for that section.

<nav> contains the main navigation functionality for the page.

<footer> represents a group of end content for a page.

Non-semantic wrappers

if you want to separate some content that has no semantic value, or you just want to use some CSS or JavaScript to it, elements like <div> or <span> could be useful. It is a good practice to give a class attribute to each element.

<span> is an inline non semantic element.

<div> is a block level non semantic element.

using too many divs can be very convenient but it is a very bad practice, as it has no meaning at all.

Line breaks and Horizontal rules

The <br> element is used for breaking a paragraph in rows. It is used as an emtyy element.

roses are red
violets are blue
I don't know HTML
neither do you.

<hr> elements create a horizontal rule in the document that denotes a thematic change in the text (such as a change in topic or scene). Visually it just looks like a horizontal line. See the example below:

Ron was backed into a corner by the marauding netherbeasts. Scared, but determined to protect his friends, he raised his wand and prepared to do battle, hoping that his distress call had made it through.

Meanwhile, Harry was sitting at home, staring at his royalty statement and pondering when the next spin off series would come out, when an enchanted distress letter flew through his window and landed in his lap. He read it hazily and sighed; "better get back to work then", he mused.
