Debugging in HTML

HTML and Debugging

Debugging is not scary

As you are new to programming, is very common to be scared when an error occurs. Those large texts in red may seem like greek for a newbie. You don't have to be scared though, because for that purpose you have Debugging tools, that helps you to fix those errors and identify them easier.

Therefore, in order to be a good programmer, a deep knowledge of the Debugging tools is as important as dominating the programming language itself.

Permissive code

HTML is a more permissive language than the traditional programming languages(like Java, C#, etc...). That means that errors can be interpret as valid code, but it may not have the output that you expected. this is mainly because html is an interpreted language, and not compiled (like the traditional ones).
