links best practices

  • don't repeat the URL as part of the link

    don't do this: https://www.microsoft.com/pt-br

  • don't say 'link' or 'links to' in the link text

    bad example: use that link to access google.

  • keep your link text as short as possible

    long link names usually annoy screen readers.

  • do not use the same word to label different links

    words like 'click here' don't mean anything, they dont give the idea of what the link really is.

  • use relative links wherever possible

    it makes your code more readable.

    it makes the browser's job more efficient.
